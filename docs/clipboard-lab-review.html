clipboard-lab

Five macOS clipboard managers built from scratch, ranked blind against five real ones by a reviewer with no memory of who built what.

The experiment: build five clipboard history managers (clip-01 through clip-05), each iteration addressing the previous one's weaknesses. Test alongside Raycast, Maccy, Alfred, Clipy, and Flycut. Anonymize all ten into identical spec sheets. Hand them to a fresh reviewer with zero context. See where ours land.

The result: ours took 1st, 2nd, and 5th. Mean rank 4.6 for ours vs 6.4 for existing. Every iteration moved up in the final ranking. The reviewer called clip-05 "the strongest clipboard-first design in the set" without knowing we built it.

The asterisk: this was a spec-sheet review, not a product review. Nobody ran any of the ten apps for more than three seconds. Keep reading.

What each iteration added

  • clip-01 — core loop: pasteboard polling, SQLite at 0600, Carbon global hotkey, 200-entry ring, concealed-type exclusion.
  • clip-02 — explicit keyboard controls (arrows / enter / escape / ⌘1-9), PNG image storage with metadata, config-driven privacy (per-app exclusions, max-bytes, pause toggle), FSEvents reload.
  • clip-03 — real AppKit settings window, query DSL (type:, app:, after:, quoted phrases), URL clips as distinct kind, 13-case parser harness.
  • clip-04 — AES-GCM payload encryption, Keychain-backed key management, NSDataDetector URL detection, broad settings (retention, polling, hotkey modifiers, auto-start).
  • clip-05 — HMAC-indexed encrypted search (no plaintext searchable text), Vision.framework on-device OCR for image clips, lazy-load history panel, debounced search, 31-case verifier, NSVisualEffectView polish, dark mode.

How the blind review worked

  • All 10 entries were distilled into identical spec-sheet format — 6 files each: summary, features, security, storage, performance, ux.
  • Letters A-J were shuffled over the 10 apps. The reviewer saw only the letters.
  • Every product name, source link, build tooling mention, and identifying string was scrubbed from the entries.
  • File mtimes were normalized so creation order couldn't leak iteration.
  • The reviewer ran in an isolated /tmp/ directory — no path back to the lab's SPEC.md or CONTEXT.md.
  • The reviewer was told: "you have no prior knowledge of these products; rank objectively; do not guess identities."

Caveats — read before celebrating

  • Spec-sheet review, not product review. The reviewer never ran any app. A clipboard manager is judged by feel over weeks of daily use. No amount of written documentation captures that.
  • Documentation advantage. Our five came with detailed threat models answering exactly the questions a security/perf reviewer asks. The existing five were summarized from public marketing pages and OSS README skims. Same doc depth on both sides would have changed the ranking.
  • Our builds are unsigned and untested at scale. Raycast and Maccy have years of real-world bug reports, Accessibility-flow polish, release tracking, and edge-case handling we haven't touched. We shipped five .app bundles that launched cleanly once.
  • One reviewer, not a panel. A different blind reviewer with different priors could invert several positions. Alfred and Clipy especially lost points for sparse public clipboard-specific docs — not because the products are weak.
  • This is a signal about iteration discipline, not a product claim. What it shows is that five rounds of build/self-review/simplify/team-review produce a codebase that a fresh reviewer reads as thorough. It does not show that clip-05 would replace Raycast in daily use. It wouldn't.

The honest headline

The iterate-then-critique loop produces impressively deep documentation and a working codebase. On paper, clip-05 outranks the strongest free existing macOS clipboard managers. In practice, the existing apps would probably win a real-world head-to-head on polish, reliability, and all the thousand things you only learn by shipping to real users.

How it was built

Claude (orchestrator) wrote the spec and held the blind-review manifest. Every build, review, simplification, and team-review pass was dispatched to Codex via codex exec --full-auto running in a sandboxed workspace. Claude held only the thin coordination state — reading short summary files, never the 10,000+ lines of Swift. Total Claude-side token cost for the experiment: well under 100k.
